Se utilizo git branch para la creación de la rama feature, se creo hicieron modificaciones en la rama principal main y fature de los archivos main.py y README.md, se realizaron 3 commits es la rama feature los cuales unimos en uno solo usando el rebase interactivo git rebase -i HEAD~3, el uso de git rebase es un proceso destructivo debido a que no dejamos registro de los 3 commit fusionado pero si permaneciron los cambios realizados en los archivos main.py y README.md. Luego nos posicionamos en la rama main para fusionar las ramas con el comando $ git merge feature, seguidamente resolvemos los conflictos y tendremos esta pequeña practica de git terminada.
